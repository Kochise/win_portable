	Device Manager 1.4 - плагин для Total Commander 5.50 и новее
	      Улучшенный Windows Device Manager(Диспетчер устройств)
___________________________________________________________________________

				 Ручная установка:

 1. Распакуйте архив в любой пустой каталог
 2. Выберите Конфигурация->Опции->Операции->FS-Pugins(WFX-плагины)
 3. Нажмите "Add"
 4. В каталоге, где были файлы были распакованы ранее, выберите devman.wfx
 5. OK.
 6. (!) Для того, чтобы использовать возможность удалений устройств,
    скопируйте devcon.exe в папку \Windows\System32\

 Плагин должен появиться в "Сетевое окружение" под именем "Device Manager"
___________________________________________________________________________

 				Преимущества плагина:

    + Показ АБСОЛЮТНО ВСЕХ установленных в системе устройств;
    + Вызов диалога свойств по нажатию ENTER;
    + Возможность создать набор драйверов для уже установленных устройств(F5);
      (резервное копирование драйверов);
    + Возможность удалять устройства по F8(Только 32BIT системы!);
    + Вызов Мастера Устранения Неполадок по нажатию Alt+Enter(Недоступно в 2000!);
    + Показ краткой информации об устройстве по нажатию F3.

 Тестировался под Windows 2000,XP. 9x/ME не поддерживается.
 Devcon.exe - 32BIT версия.
 Вы должны иметь права Администратора.

				     F.A.Q.

 Q: Почему все иконки какие-то серенькие? На моей цветовой схеме(серые панели)
    их почти не видно.Можно ли сделать их поярче?
 А: Можно, но я так не сделаю, т.к. эти иконки не идут с плагином, а берутся
    прямо из системы, для экономии размера плагина.Попробуйте другую цветовую
    схему.

 Q: Можно ли добавить поддержку Win98? Просто не все пользуются XP.
 A: К сожалению поддержка 98/ME не возможна. Это не моя прихоть, а
    специфика ОС.Единственное, что возможно под 98 - это только показ списка 
    устройств.Открытие диалога свойств устройства, создание набора драйверов,
    и т.д. - невозможно.В Windows SDK про это нет ни слова, а экспериментально
    я разобраться не смог.Поэтому 98 и не поддерживается. Возможно, появится
    версия под 98/ME, но из всех функций плагина останется только резервное
    копирование драйверов(IMHO самая важная функция).

 Q: А зачем вообще введена возможность создания пакета драйверов? У меня драйвер
    вон на болванке от производителя лежит, и я его всегда оттуда устанавливаю.
 A: Нужно это прежде всего для администраторов. Можно собрать в одну папку все
    драйвера, нужные системе, и не лазить в поисках болванки с родным драйвером
    (что особенно актуально для компьютерных клубов,больших офисов, и т.д., где 
    CD-ROM на каждую машину не поставишь).К тому же, некоторые производители 
    оборудования оформляют драйвера в виде exe-файлов, с которыми также 
    устанавливаются всякие дополнительные фишки,не всегда нужные(IMHO). Например,
    Creative EAX Demo,Acer On-line manual, и т.п.

 Q: Почему при просмотре информации об устройстве по нажатию F3 показано одно
    количество драйверов, а при копировании по F5 копируется меньшее количество?
 A: Потому что по F3 показаны все файлы драйвера, нужные устройству(и для XP, и
    для 98), а по F5 копируются только установленные в системе файлы(т.е. для XP).

 Q: Почему для моей карты Creative SB LIVE! 5.1 стандартный Диспетчер устройств
    Windows показывает файлы, которых нет в пакете драйверов?
 A: Некоторые файлы поддержки драйверов звуковых карт поставляются с Windows.
    Например, ks.sys. Особенно это заметно для интегрированных звуковых карт.

 Q: Что-то у плагин у меня не работает :(.
 A: Пишите мне на e-mail с подробным описанием проблемы, и ждите ответа.
