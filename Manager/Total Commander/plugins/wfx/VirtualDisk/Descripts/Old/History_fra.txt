Virtual Disk, Version 1.30 Finale
  Module Système de fichiers pour Total Commander qui permet de monter des images de disques en lecteurs virtuels supplémentaires dans le système.

Auteur du module : Konstantin Vlasov, 2009
Page Internet : http://flint-inc.ru/
Courriel :   support@flint_inc.ru

Le pilote est basé sur le projet sources-libres “FileDisk” de Bo Branten, 2004
Page Internet : http://www.acc.umu.se/~bosse/
Courriel :   bosse@acc.umu.se


    /============================\
    |      Évolution des versions :      |
    \============================/

Version 1.30 Finale (compilation 3001) :
  1. Système de numérotation des versions légèrement amélioré.
  2. Ajout de messages décrivants certaines erreurs avec plus de détails.
  3. Ajout de la section “FAQ” aux fichiers “ReadMe…” .

Version 1.30 beta 3 :
  1. Le remontage automatique des disques après redémarrage est implémenté.
  2. Élimination d'une bogue qui faisait que les images démontées par un redémarrage apparaissaient toujours montées.
  3. Ajout de pilotes 64 bits : un testé pour Windows x64 et un expérimental fpour Windows 64 bits (Intel Itanium).
  4. Deux fichiers de langue Allemand ont été ajoutés (de Frank Bultge et de Peter Brockamp).
  5. Plusieurs petites réparations….

Version 1.30 beta 2 :
  1. Le pilote est remanié de façon à ce que le module n'exige plus les droits d'administrateur pour fonctionner.
     Ils ne sont requis que pour installer le pilote. 
  2. À côté de la nouvelle méthode d'installation du pilote (au moyen du fichier INF), l'ancienne - qui demande de copier le fichier du pilote et d'importer les données dans le registre - a été rétablie, car la nouvelle méthode ne convenait pas à certains systèmes.
  3. Les fichiers NRG sont désormais ajoutés par défaut au module en mode CD¦DVD, vu que dans certains cas, ce format peut être monté et correctement reconnu par le système.
  4. La version 64 bits du pilote “VD_FileDisk” a été ajoutée à titre expérimental.

Version 1.30 beta 1 :
  1. Ajout du mode d'émulation de lecteur de disquette.
  2. Le pilote a été renommé de «FileDisk» en “VD_FileDisk” évitant ainsi les conflits avec d'autres programmes utilisant un pilote similaire (par exemple, WinImage 7.0+).
  3. L'installation du pilote est simplifiée grâce à un fichier INF.
  4. Ajout : traduction d'interface en Polonais (Pawel Wawrzyszko).
  5. Ajout : traduction d'interface en Ukrainien (Butsky Igor).

Version 1.26 :
  1. Élimination d'une bogue quand le nom de la boîte de groupe “Mode de montage” n'était pas traduit.
  2. Changé : police interne de l'horrible MS Sans Serif en MS Shell Dlg “virtuelle” qui peut donc être facilement changée
      sur chaque système.
  3. Ajout : traductions d'interface, Français (Claude Charries) et Espagnol (Luis Mejia).

Version 1.25 :
  1. Le module utilise désormais des fichiers *.LNG pour changer sa langue. La langue peut se choisir en ouvrant la boîte “À propos… ” (entrée «Propriétés» du menu local ou Alt+Entrée sur «Virtual Disks») —> bouton Agencements. Il y a deux langues dans le paquetage : Russe et Anglais (Anglais par défaut).
  2. Ajout des lettres de raccourcis dans les dialogues pour se déplaver rapidement entre les contrôles (lettres soulignées… Attention à la config. par défaut de Win XP !).
  3. L'exécutable “Convert.exe” est retiré du paquetage, il n'était nécessaire que pour les plus anciennes versions.
  4. Ajout :  fichier “pluginst.inf” pour installation automatique dans  6.5 et +.

Version 1.20 :
  1. Cette version utilise le pilote “filedisk.sys”mis à jour de Bo Brenten (1.0.0.13). 
      Les précédentes versions employées dans le module avaient une bogue provoquant des fuites de mémoire. 
     Elles rendaient le système de plus en plus lent, jusqu'à plantage complet. 
     Également, ce pilote fonctionne mieux avec de grosses images de cédérom (plus de 700 Mio).

Version 1.10 :
  1. Réécriture de la totalité des codes. Beaucoup d'éliminations de bogues en résultèrent, ainsi qu'un travail plus efficace (liste de fichiers) et meilleur (avec plusieurs instances de TC).
  2. le type BIN se comporte maintenant comme un fichier ISO, car il est reconnu par Windows.
  3. Ajout :  contrôle du type de fichier lors de la copie.
  4. Éliminées : plusieurs bogues de copie¦effacement de fichiers.
  5. Il n'est désormais pas permis d'effacer une image montée sans la démonter.

Version 1.00 :
  La première version . Elle peut monter des images aux formats ISO, BIN (en mode CD) et IMG (en modes RW (réinscriptible) et  RO (Lecture seule).
